# LoL AI 분석 플랫폼 확장 기획서

> Riot API 기반 전적/타임라인 분석에 **플레이어 능력치 평가**, **랭크 포지션 분석**, **AI 리포트**, 그리고 확장 기능으로 **하이라이트·비전 기반 복기 분석**을 결합한 League of Legends 분석 플랫폼 기획서.

---

## 1. 프로젝트 개요

### 1.1 프로젝트 목표

기존 전적 검색 서비스처럼 단순히 KDA, 승률, CS, 딜량만 보여주는 것이 아니라, 사용자의 최근 랭크 경기 데이터를 기반으로 다음을 분석한다.

- 경기 흐름 분석
- 승률 예측
- 플레이어 능력치 평가표 생성
- 포지션별 적합도 분석
- 반복되는 패배 패턴 탐지
- AI 기반 개선 방향 리포트 생성
- 확장 기능으로 주요 장면 하이라이트 추출 및 비전 분석

최종적으로는 사용자가 전적을 검색했을 때 다음과 같은 질문에 답할 수 있는 서비스를 목표로 한다.

> “나는 어느 라인에 설 때 가장 잘하는가?”  
> “내가 지는 경기는 보통 어떤 패턴에서 발생하는가?”  
> “라인전은 괜찮은데 왜 게임을 못 굴리는가?”  
> “교전, 맵리딩, 오브젝트 판단 중 무엇이 약한가?”  
> “내 데스는 단순 실수인가, 운영상 필요한 압박이었는가?”

## 2. 핵심 컨셉

### 2.1 서비스 한 줄 설명

> Riot API의 정량 데이터로 경기 흐름과 플레이 성과를 분석하고, 하이라이트 클립 기반 Vision AI를 확장 기능으로 활용해 포지셔닝·라인관리·맵리딩처럼 수치 데이터만으로 보기 어려운 플레이 능력까지 보완 평가하는 LoL AI 분석 플랫폼.

### 2.2 핵심 차별점

일반 전적 검색 서비스는 결과 중심 지표를 보여준다.

- 승률
- KDA
- CS
- 딜량
- 시야 점수
- 아이템 빌드

본 프로젝트는 여기서 더 나아가 **플레이 습관과 판단 능력**을 점수화한다.

- 라인전 능력
- 교전 능력
- 오브젝트 운영
- 맵리딩 / 위험 인식
- 리드 전환 능력
- 안정성 / 던짐 방지
- 포지션별 적합도
- 크로스맵 압박 판단
- 하이라이트 기반 복기 분석

## 4. MVP 범위와 확장 범위

### 4.1 MVP에서 우선 구현할 기능

MVP는 **Riot API로 제공되는 정량 데이터 기반 분석**에 집중한다.

```text
MVP 핵심 기능

1. Riot ID 기반 소환사 검색
2. 최근 랭크 경기 목록 조회
3. 경기 상세 데이터 저장
4. 타임라인 기반 피처 추출
5. 규칙 기반 승률 예측
6. 플레이어 능력치 평가표
7. 랭크 포지션 분석 탭
8. AI 종합 리포트 생성
```

MVP에서는 Vision AI를 필수 기능으로 넣지 않는다. 대신 확장 가능하도록 데이터 구조와 UI 영역을 미리 고려한다.

### 4.2 확장 기능으로 구현할 기능

비전 분석은 구현 난이도가 높고, 영상/리플레이/프레임 처리까지 필요하기 때문에 2차 확장 기능으로 분리한다.

```text
확장 기능

1. 주요 장면 자동 추출
2. 영상 업로드 기반 클립 생성
3. 프레임 샘플링
4. Vision AI 기반 장면 분석
5. 미니맵 위험 인식 분석
6. 한타 포지셔닝 분석
7. 라인 웨이브 상태 분석
8. 카이팅 / 거리 조절 분석
9. 크로스맵 압박 판단 보정
10. 고급 확장: 로컬 에이전트 기반 자동 녹화 및 하이라이트 생성
```

## 6. 플레이어 종합 평가표

### 6.1 공통 능력치

모든 포지션에 공통적으로 적용되는 평가 항목이다.

| 평가 항목 | 설명 | 주요 데이터 |
|---|---|---|
| 라인전 능력 | 초반 성장과 주도권 확보 능력 | CS@10, Gold@10, XP@10, 솔로킬/솔로데스 |
| 교전 능력 | 킬 관여, 딜 기여, 한타 영향력 | 킬 관여율, 피해량, 받은 피해, 한타 전후 골드 변화 |
| 오브젝트 운영 | 드래곤/전령/바론 전후 판단 능력 | 오브젝트 전후 킬/데스, 오브젝트 획득률 |
| 맵리딩 / 위험 인식 | 상대 위치와 아군 위치를 고려한 위험 회피 능력 | 고립 데스, 시야 점수, 오브젝트 전 사망 |
| 리드 전환 능력 | 초반 우위를 타워/오브젝트/승리 조건으로 연결하는 능력 | 골드 우위 이후 오브젝트/타워 전환 |
| 안정성 | 중요한 순간에 손해가 큰 데스를 줄이는 능력 | Death Cost, Throw Index, 제압골 헌납 |

### 6.2 평가표 예시

```text
플레이어 종합 평가표

라인전 능력        78점  B+
교전 능력          64점  C+
오브젝트 운영      82점  A-
맵리딩             61점  B-
리드 전환          70점  B
안정성             48점  D+

AI 요약:
초반 라인전과 오브젝트 준비는 좋은 편이지만,
중반 이후 사이드 운영 중 고립 데스가 반복되어 안정성 점수가 낮습니다.
```

## 7. 핵심 커스텀 지표 설계

### 7.1 Death Cost Score

단순 데스 수가 아니라, **그 데스가 팀에 얼마나 큰 손해를 줬는지** 평가한다.

```text
Death Cost Score 산정 요소

- 오브젝트 직전 사망 여부
- 사망 후 90초 이내 드래곤/바론/전령 손실 여부
- 사망 후 타워 손실 여부
- 제압골 헌납 여부
- 20분 이후 사망 여부
- 사망 당시 팀의 골드 우위 여부
```

예시 리포트:

```text
전체 데스 수는 5회로 평균 수준이지만,
그중 3회가 오브젝트 직전 90초 이내에 발생해 팀 손실로 크게 이어졌습니다.
```

### 7.2 Throw Index

유리한 게임을 스스로 불리하게 만든 장면을 수치화한다.

```text
Throw Index 상승 조건

- 골드 우위 상황에서 연속 데스
- 바론/드래곤 직전 고립 데스
- 바론 획득 후 바로 사망
- 제압골 헌납
- 한타 패배 후 골드 역전
```

예시 리포트:

```text
초반 리드 확보율은 높지만,
20분 이후 오브젝트 직전 데스가 반복되어 Throw Index가 높게 나타났습니다.
```

### 7.3 Objective Setup Score

오브젝트를 먹었는지만 보는 것이 아니라, **먹기 전 준비가 잘 되었는지** 평가한다.

```text
Objective Setup Score 산정 요소

- 오브젝트 90초 전 시야 점수 변화
- 오브젝트 직전 아군 사망 여부
- 정글러 생존 여부
- 라인 주도권 여부
- 오브젝트 전 교전 승리 여부
- 오브젝트 이후 골드 변화
```

예시 리포트:

```text
팀은 골드가 앞서 있었지만,
드래곤 생성 전 90초 동안 시야 확보와 합류가 부족해 오브젝트 전환율이 낮았습니다.
```

### 7.4 Lead Conversion Rate

라인전에서 만든 리드를 실제 승리 조건으로 전환했는지 평가한다.

```text
Lead Conversion Rate 산정 요소

- 10분 골드 / CS / XP 우위
- 15~25분 사이 타워 획득 여부
- 전령/드래곤/바론 연결 여부
- 초반 킬 이후 오브젝트 전환 여부
- 리드 상황에서 사망으로 이득을 잃은 횟수
```

예시 리포트:

```text
라인전 리드는 있었지만,
중반 오브젝트 참여와 사이드 안정성이 낮아 초반 우위를 승리 조건으로 전환하지 못했습니다.
```

### 7.5 Visible Threat Response Score

상대 위치가 보였거나 예측 가능한 상황에서 제대로 반응했는지 평가한다.

```text
Visible Threat Response Score 산정 요소

- 상대 정글/미드 위치가 드러난 뒤 사망 여부
- 위험 지역 진입 전 아군 위치
- 오브젝트 전 시야 없는 지역 진입 여부
- 라인 압박 중 후퇴 타이밍
- 고립 상태에서의 생존 여부
```

MVP에서는 API 기반 근사치로 계산하고, 확장 단계에서는 미니맵 비전 분석으로 보정한다.

### 7.6 Cross-map Pressure Score

사이드 압박이나 미끼 플레이가 실제 팀 이득으로 연결되었는지 평가한다.

```text
Cross-map Pressure Score 산정 요소

- 바론/드래곤 반대편 라인 압박 여부
- 상대 인원을 몇 명 끌어냈는지
- 그 사이 아군이 오브젝트를 획득했는지
- 플레이어가 죽었더라도 팀 전체 이득이 컸는지
- 제압골 헌납 여부
```

판정 예시:

```text
전략적 크로스맵 압박 성공:
플레이어는 바론 반대편 바텀 라인에서 상대 2명을 끌어냈고,
그 사이 아군이 바론을 획득했습니다.
사망은 발생했지만 오브젝트 교환 가치가 높아 전략적 압박 성공으로 평가됩니다.
```

## 8. 랭크 포지션 분석 탭

### 8.1 기능 목적

사용자의 최근 랭크 게임을 포지션별로 분리해 분석하고, 어느 라인에 설 때 가장 좋은 퍼포먼스를 보이는지 평가한다.

```text
랭크 분석 탭 목표

- 포지션별 승률 확인
- 포지션별 핵심 능력치 비교
- 공통 능력치와 라인 전용 능력치 분리
- 추천 포지션 산출
- 라인별 약점과 개선 방향 제시
```

### 8.2 화면 예시

```text
랭크 분석

[전체] [탑] [정글] [미드] [원딜] [서포터]

추천 포지션: 미드 / 원딜
주의 포지션: 정글

포지션 적합도
탑      71점
정글    54점
미드    82점
원딜    78점
서포터  63점
```

### 8.3 포지션 점수 계산 방식

승률만으로 포지션 적합도를 판단하지 않는다. 표본 수가 적은 포지션의 과대평가를 막기 위해 신뢰도를 함께 표시한다.

```text
포지션 적합도 =
승률 점수
+ 라인별 핵심 지표
+ 공통 운영 지표
+ 최근 폼
+ 표본 수 신뢰도 보정
```

예시:

```text
미드 적합도 =
승률
+ 라인전 안정성
+ 로밍 영향력
+ 오브젝트 합류
+ 교전 능력
+ 맵리딩
```

```text
원딜 적합도 =
승률
+ 성장 능력
+ DPS 효율
+ 포지셔닝 안정성
+ 카이팅 분석
+ 후반 캐리력
```

## 9. 라인별 평가 설계

### 9.1 탑 평가

탑은 라인전, 사이드 운영, 텔레포트 활용, 크로스맵 압박이 중요하다.

| 평가 항목 | 설명 |
|---|---|
| 라인전 주도권 | CS@10, Gold@10, XP 차이, 솔로킬/솔로데스 |
| 사이드 운영 | 20분 이후 사이드 압박, 고립 데스 관리 |
| 텔레포트 활용 | 오브젝트/한타 합류, 텔레포트 이후 이득 연결 |
| 스플릿 푸시 가치 | 타워 압박, 상대 인원 분산, 크로스맵 이득 |
| 안정성 | 사이드에서 죽은 뒤 오브젝트 손실 여부 |

예시 리포트:

```text
라인전 CS 우위는 안정적으로 가져가는 편이지만,
20분 이후 사이드 압박 중 상대 정글/미드 위치가 보인 상태에서도 후퇴가 늦어지는 장면이 반복됩니다.

다만 바론 반대편 사이드에서 상대 2명을 끌어낸 뒤 아군 바론 획득으로 연결된 장면이 있어,
크로스맵 압박 능력은 좋은 편입니다.
```

### 9.2 정글 평가

정글은 성장 속도, 갱킹 영향력, 오브젝트 컨트롤, 동선 효율이 중요하다.

| 평가 항목 | 설명 |
|---|---|
| 정글링 속도 / 성장 | 초반 CS, 레벨링, 상대 정글과 골드/경험치 차이 |
| 갱킹 영향력 | 갱킹 후 킬/어시, 스펠 교환, 라인 이득 |
| 오브젝트 컨트롤 | 드래곤/전령/바론 관여, 오브젝트 전 준비 |
| 동선 효율 | 캠프 손실과 갱킹 성공의 균형 |
| 라인 커버 | 다이브 방어, 역갱, 라인 손실 방지 |

예시 리포트:

```text
초반 정글링 속도는 평균 이상이지만,
갱킹 성공 후 오브젝트로 전환하는 비율이 낮습니다.

특히 드래곤 생성 전 60초 동안 바텀 주도권이 없는 상태에서 드래곤을 시도해,
교전 손실로 이어진 장면이 확인되었습니다.
```

### 9.3 미드 평가

미드는 라인관리, 로밍, 정글/오브젝트 연계, 전 맵 영향력이 중요하다.

| 평가 항목 | 설명 |
|---|---|
| 라인전 안정성 | CS@10, Gold@10, XP 차이, 솔로킬/솔로데스 |
| 로밍 영향력 | 로밍 후 킬/어시, 오브젝트 연결 |
| 라인 관리 | 웨이브를 밀고 움직였는지, 로밍 손실 관리 |
| 오브젝트 합류 | 전령/드래곤 타이밍 합류, 강가 교전 참여 |
| 중후반 사이드 운영 | 사이드 배치 안정성, 합류 타이밍 |

예시 리포트:

```text
로밍 빈도는 높지만, 로밍 전 웨이브 정리가 부족해 본인 미드 CS 손실이 큽니다.
성공한 로밍은 바텀 교전 2회였고, 그중 1회는 드래곤 획득으로 연결되었습니다.
```

### 9.4 원딜 평가

원딜은 성장, DPS, 카이팅, 포지셔닝 안정성이 중요하다.

| 평가 항목 | 설명 |
|---|---|
| 성장 능력 | CS@10, CS@20, 골드 수급, 아이템 타이밍 |
| DPS 기여 | 챔피언 피해량, 골드 대비 피해량, 한타 피해 기여 |
| 카이팅 / 거리 조절 | 교전 중 거리 유지, 공격 후 후퇴 움직임 |
| 포지셔닝 안정성 | 먼저 물리는 빈도, 한타 시작 위치 |
| 후반 캐리력 | 25분 이후 한타 기여, 바론/장로 교전 생존 |

예시 리포트:

```text
성장 지표는 좋고 20분 CS도 안정적이지만,
주요 한타 3개 중 2개에서 앞라인보다 앞에 위치한 장면이 확인되었습니다.

딜 기대치는 높지만 포지셔닝 리스크 때문에 실제 한타 생존 시간이 짧아,
DPS 효율이 낮게 나타났습니다.
```

### 9.5 서포터 평가

서포터는 시야 장악, 로밍, 이니시, 보호, 오브젝트 세팅이 중요하다.

| 평가 항목 | 설명 |
|---|---|
| 시야 장악 | 시야 점수, 제어 와드, 와드 제거, 오브젝트 전 시야 |
| 로밍 영향력 | 미드/정글 교전 합류, 로밍 후 킬/어시 |
| 라인전 보조 | 바텀 2:2 기여, 원딜 성장 보호 |
| 이니시 / 받아치기 | 한타 시작 기여, 무리한 이니시 여부 |
| 보호 능력 | 원딜 생존 보조, 암살자 진입 차단 |

예시 리포트:

```text
시야 점수는 평균 이상이지만, 드래곤 생성 전 강가 시야를 미리 잡는 비율은 낮습니다.
로밍 성공률은 높고 미드 교전에 자주 관여하지만,
로밍 중 바텀 원딜이 압박받는 장면이 일부 확인되었습니다.
```

## 10. 맵리딩 / 운영 판단 분석

### 10.1 분석 목적

맵리딩 분석은 단순히 “죽었는가”를 보는 것이 아니라, 다음을 구분하는 것을 목표로 한다.

```text
1. 상대 위치를 볼 수 있었는데도 당한 회피 가능한 데스
2. 상대를 일부러 끌어내 팀에게 이득을 만든 전략적 압박
3. 의도는 있었지만 팀 상황과 맞지 않아 실패한 운영
```

### 10.2 판정 유형

#### A. 회피 가능한 고립 데스

```text
조건 예시

- 상대 위치가 드러났거나 예측 가능함
- 아군이 멀리 있음
- 근처에 연결 가능한 오브젝트 이득이 없음
- 라인을 과하게 밀었음
- 사망 후 상대가 오브젝트/타워를 획득함
```

판정:

```text
회피 가능한 고립 데스
```

#### B. 전략적 크로스맵 압박 성공

```text
조건 예시

- 아군이 바론/드래곤 근처에 있음
- 본인은 반대편 사이드 라인을 압박함
- 상대 2명 이상이 본인 쪽으로 반응함
- 그 사이 아군이 오브젝트를 획득함
- 죽더라도 팀 전체 이득이 더 큼
```

판정:

```text
전략적 크로스맵 압박 성공
```

#### C. 크로스맵 압박 시도 실패

```text
조건 예시

- 반대편 사이드 압박 구도는 맞음
- 그러나 아군이 오브젝트를 시작할 준비가 안 됨
- 플레이어가 너무 빨리 사망함
- 상대를 충분히 묶지 못함
- 결과적으로 오브젝트 이득 없이 제압골만 헌납함
```

판정:

```text
의도는 있었으나 실행 실패
```

### 10.3 리포트 예시

```text
장면: 22:15 바텀 사이드 압박

분석:
- 바론이 살아 있는 상태에서 아군 4명이 바론 근처에 위치
- 플레이어는 바텀 2차 타워 쪽 라인을 압박
- 상대 미드와 정글이 바텀으로 반응
- 18초 뒤 아군이 바론 획득
- 플레이어는 사망했지만 상대 주요 인원을 바론 반대편으로 끌어냄

판정:
전략적 크로스맵 압박 성공

점수 영향:
운영 판단 +10
오브젝트 연계 +12
안정성 -2
```

## 11. AI Highlight Analyzer 확장 기능

### 11.1 기능 목적

하이라이트 기능은 단순히 멋있는 장면을 저장하는 것이 아니라, **승패에 영향을 준 주요 장면을 자동으로 찾아 분석하는 기능**이다.

```text
일반 하이라이트:
킬함 → 클립 저장
바론 먹음 → 클립 저장

본 프로젝트의 하이라이트:
킬함 → 왜 좋은 플레이였는지 분석
죽음 → 이 데스가 얼마나 손해였는지 분석
오브젝트 전 사망 → Death Cost와 연결
한타 패배 → 포지셔닝/미니맵/합류 상태 분석
```

### 11.2 처리 흐름

```text
1. Timeline API로 주요 이벤트 탐지
   - 킬
   - 데스
   - 드래곤/전령/바론
   - 타워 파괴
   - 골드 차이 급변
   - 오브젝트 직전 사망

2. 이벤트별 Highlight Score 계산

3. 사용자가 업로드한 경기 영상에서 해당 시간 앞뒤 15~20초 추출

4. 클립에서 프레임 샘플링
   - 시작 시점
   - 교전 직전
   - 스킬 교환 시점
   - 사망/킬 직전
   - 결과 시점

5. Vision AI로 장면 분석

6. AI 에이전트가 최종 복기 리포트 생성
```

### 11.3 Highlight Score

```text
Highlight Score =
킬 가치
+ 오브젝트 연결 여부
+ 제압골 여부
+ 한타 참여 인원
+ 골드 차이 변화량
+ 데스 비용
+ 승률 변화량
```

예시:

```text
이번 경기 주요 장면 TOP 3

1. 21:42 바론 앞 한타 승리
   - Highlight Score: 92
   - 승률 예측 변화: +18%
   - 바론 획득으로 연결됨

2. 16:10 드래곤 직전 사망
   - Highlight Score: 84
   - Death Cost Score: 높음
   - 사망 후 40초 이내 드래곤 손실

3. 12:34 솔로킬
   - Highlight Score: 68
   - 라인 주도권 확보
```


### 11.4 고급 확장: Local Highlight Agent

사용자가 직접 전체 경기 영상을 업로드하지 않아도, 추후에는 로컬 PC에서 동작하는 데스크톱 에이전트가 경기 중 화면을 녹화하고 주요 이벤트 시간을 기록하는 방식으로 자동 하이라이트 기능을 확장할 수 있다.

이 기능은 피시방이나 전적 앱에서 제공하는 자동 하이라이트 저장 기능과 유사한 방향이지만, 본 프로젝트에서는 단순 저장이 아니라 **경기 후 복기와 AI 분석을 위한 장면 수집 도구**로 설계한다.

```text
Local Highlight Agent 처리 흐름

1. 사용자가 로컬 에이전트 실행
2. 게임 시작 감지
3. 화면 녹화 시작
4. Live Client Data 또는 로컬 이벤트 정보를 기반으로 주요 시점 기록
   - 킬
   - 데스
   - 어시스트
   - 드래곤/전령/바론
   - 타워 파괴
   - 한타 후보 시점
5. 경기 종료 후 이벤트 시간 기준으로 앞뒤 15~20초 클립 자동 생성
6. 생성된 클립을 서버에 업로드하거나 로컬에서 분석 대기
7. 서버의 Vision AI 분석 및 AI 복기 리포트와 연결
```

MVP와의 차이는 다음과 같다.

| 구분 | MVP / 1차 확장 | 고급 확장 |
|---|---|---|
| 영상 입력 | 사용자가 직접 영상 업로드 | 로컬 에이전트가 자동 녹화 |
| 하이라이트 생성 | Timeline 이벤트 + 업로드 영상 기준으로 클립 추출 | 로컬 이벤트 트리거 기반 자동 클립 생성 |
| 구현 형태 | 웹 서비스 중심 | 데스크톱 앱 또는 로컬 에이전트 필요 |
| 사용 목적 | 경기 후 복기 | 경기 후 자동 복기 편의성 향상 |

중요한 점은 이 기능도 **실시간 조언 도구가 아니라 경기 후 분석 도구**로 유지해야 한다는 것이다. 경기 중 “지금 빠져라”, “바론을 쳐라”처럼 사용자의 판단을 직접 지시하지 않고, 게임이 끝난 뒤 자동으로 생성된 클립을 분석해 복기 리포트에 반영한다.

## 12. Vision AI 확장 분석

### 12.1 Vision 기능은 확장으로 분리하는 이유

비전 분석은 다음 이유로 MVP 필수 기능에서 제외한다.

```text
- Riot API만으로 영상 프레임을 직접 얻는 구조가 아님
- 사용자가 직접 클립/스크린샷/녹화 영상을 제공해야 함
- 영상 처리와 프레임 추출이 필요함
- 챔피언, 스킬 이펙트, 미니맵 아이콘 인식 난이도가 높음
- 분석 결과에 불확실성이 있으므로 신뢰도 표시가 필요함
```

따라서 MVP에서는 정량 데이터 기반 분석을 완성하고, 이후 Vision 분석을 보조 기능으로 추가한다.

### 12.2 Vision으로 보완할 수 있는 능력

| 분석 영역 | API 기반 한계 | Vision 보완 가능성 |
|---|---|---|
| 맵리딩 | 실제 화면에서 적이 보였는지 확인 어려움 | 미니맵 프레임으로 적 노출 여부 확인 |
| 라인관리 | 웨이브 상태를 직접 알기 어려움 | 미니언 위치로 푸쉬/프리징/빅웨이브 추정 |
| 교전 능력 | 왜 딜을 못 넣었는지 파악 어려움 | 포지셔닝, 거리 조절, 고립 여부 분석 |
| 카이팅 | 마우스 입력이나 세부 움직임 정보 없음 | 클립에서 공격 후 이동 패턴 근사 분석 |
| 오브젝트 준비 | 자리잡기와 시야 구도 파악 어려움 | 강가/부쉬/오브젝트 주변 화면 분석 |

### 12.3 Vision 분석 후보

#### 1. 미니맵 위험 인식 분석

```text
분석 대상

- 사망 전 미니맵에 상대가 보였는가
- 아군과 거리가 멀었는가
- 시야 없는 지역으로 진입했는가
- 위험 핑이나 오브젝트 상황이 있었는가
```

#### 2. 라인 웨이브 상태 분석

```text
분석 대상

- 웨이브가 상대 타워 쪽으로 밀렸는가
- 빅웨이브가 쌓였는가
- 로밍 전 라인을 밀었는가
- 다이브 위험 웨이브였는가
```

#### 3. 한타 포지셔닝 분석

```text
분석 대상

- 딜러가 앞라인보다 앞에 있었는가
- 상대 진입 챔피언과 거리가 가까웠는가
- 팀이 분산되어 싸웠는가
- 오브젝트 지형을 활용했는가
```

#### 4. 카이팅 / 거리 조절 분석

```text
분석 대상

- 공격 이후 뒤로 이동했는가
- 근접 챔피언과 거리를 유지했는가
- 스킬샷을 피하려는 움직임이 있었는가
- 벽 쪽으로 몰렸는가
```

### 12.4 Vision 결과 표현 방식

Vision 분석은 절대적인 판정으로 표현하지 않고, **가능성 + 근거 + 신뢰도**로 표현한다.

나쁜 표현:

```text
이 플레이어는 일부러 죽었습니다.
이 플레이어는 맵을 안 봤습니다.
```

좋은 표현:

```text
이 장면은 크로스맵 압박 의도가 있었을 가능성이 높습니다.
상대 2명을 바텀으로 끌어냈고, 아군 바론 획득으로 연결되어 전략적 가치가 높았습니다.
```

```text
사망 7초 전 상대 정글러가 미니맵에 노출된 것으로 보이며,
이후 후퇴 움직임이 늦어 회피 가능한 갱킹 사망으로 분류됩니다.
```

## 13. AI 에이전트 리포트

### 13.1 역할

AI 에이전트는 점수를 직접 계산하는 주체가 아니라, 백엔드에서 계산된 지표와 근거를 바탕으로 사용자에게 이해하기 쉬운 리포트를 생성한다.

```text

- 최근 경기 요약
- 포지션별 점수
- 공통 능력치 점수
- 주요 패턴
- Death Cost / Throw Index / Lead Conversion 등 커스텀 지표
- Vision 분석 결과가 있으면 해당 결과
```

```text
AI 에이전트 출력

- 강점 요약
- 약점 요약
- 반복되는 패배 패턴
- 추천 포지션
- 라인별 개선 방향
- 주요 장면 복기 설명
```

### 13.2 리포트 예시

```text
AI 종합 평가

미드에서는 로밍 후 오브젝트 연결률이 높고,
원딜에서는 성장 지표와 후반 딜 기여도가 좋은 편입니다.
```

## 19. 주의사항 및 한계

### 19.1 의도 확정 금지

AI가 플레이어의 의도를 단정하면 안 된다.

```text
피해야 할 표현
- 일부러 죽었다
- 맵을 안 봤다
- 무조건 잘못된 판단이다
```

```text
권장 표현
- 크로스맵 압박 의도가 있었을 가능성이 있습니다.
- 결과적으로 오브젝트 이득으로 연결되지 않아 손해 판단에 가깝습니다.
- 미니맵상 위험 정보가 있었던 것으로 보이며, 후퇴 반응이 늦었습니다.
```

### 19.2 신뢰도 표시

점수는 항상 근거와 신뢰도를 함께 제공한다.

```text
예시

맵리딩: 64점 / 신뢰도 보통
- 회피 가능한 갱킹 사망 2회
- 성공적인 크로스맵 압박 1회
- 영상 분석 데이터 없음
```

### 19.3 실시간 조언 제외

본 프로젝트는 경기 중 실시간으로 의사결정을 지시하는 도구가 아니라, **경기 후 복기와 분석을 위한 서비스**로 설계한다.

```text
허용 방향
- 경기 후 분석
- 자신의 경기 복기
- 최근 랭크 패턴 분석
- 하이라이트 기반 피드백

피해야 할 방향
- 경기 중 “지금 빼라” 같은 실시간 판단 지시
- 게임 클라이언트에 없는 정보를 이용한 실시간 이득 제공
- 로컬 에이전트를 실시간 코칭 도구처럼 사용하는 방향
```

## 20. 최종 프로젝트 설명 문장

> 최근 랭크 경기 데이터를 기반으로 탑, 정글, 미드, 원딜, 서포터별 플레이 성향과 능력치를 분리 평가하고, Riot API 기반 정량 지표와 AI 리포트를 결합해 사용자에게 가장 적합한 포지션과 개선 방향을 추천하는 LoL AI 분석 플랫폼이다. 이후 확장 기능으로 하이라이트 클립 기반 Vision AI 분석을 추가하여 포지셔닝, 맵리딩, 라인관리, 카이팅처럼 수치 데이터만으로 보기 어려운 플레이 능력까지 보완 평가한다. 장기적으로는 로컬 에이전트 기반 자동 녹화와 하이라이트 생성 기능을 추가해, 사용자가 직접 영상을 자르지 않아도 경기 후 주요 장면을 자동 복기할 수 있도록 확장한다.

## 21. Codex 작업 지시용 요약

```text
프로젝트 방향:
LoL 전적 검색 + 타임라인 분석 + 포지션별 능력치 평가 + AI 리포트 플랫폼.
Vision 분석은 MVP가 아니라 확장 기능으로 분리한다.

MVP 우선 기능:
1. Riot ID 기반 검색
2. 최근 랭크 경기 조회
3. 매치/타임라인 데이터 저장
4. 커스텀 지표 계산
   - Death Cost Score
   - Throw Index
   - Objective Setup Score
   - Lead Conversion Rate
5. 플레이어 종합 평가표
6. 랭크 포지션 분석 탭
7. AI 리포트 생성

확장 기능:
1. 하이라이트 후보 자동 추출
2. 영상 업로드 기반 클립 생성
3. Vision AI 기반 미니맵/포지셔닝/카이팅/라인관리 분석
4. 장기 확장으로 로컬 에이전트 기반 자동 녹화 및 하이라이트 생성
```
