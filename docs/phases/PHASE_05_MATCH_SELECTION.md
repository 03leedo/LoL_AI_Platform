# Phase 5 — Match Selection

## Read
- `docs/domain/ANALYSIS_RULES.md`
- profile output docs

## Outcome
Select representative, best-performance, and profile-deviation matches with reasons and evidence windows.

## Method
- choose comparable matches;
- standardize profile features by cohort;
- exclude remakes, short matches, off-role and critical missing data;
- use a clear distance/score method;
- store version, driving dimensions, and evidence windows.

## Acceptance
Selections are deterministic, explainable, handle ties/missing dimensions, and never label deviation as automatically worst.

Stop after this phase.
